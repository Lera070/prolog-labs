# Лабораторная работа 2: Списки на языке Prolog

**Цель работы:** Приобрести навыки рекурсивной обработки рекурсивных структур данных; научиться интерпретировать (переводить на естественный язык) Prolog-программы.

В рамках данной лабораторной работы я реализовала два отношения для работы со списками на языке Prolog в виде фактов и правил. Для каждого отношения были написаны рекурсивные алгоритмы, составлены декларативная и процедурная интерпретации, а также проведено комплексное тестирование с учетом краевых случаев.

## Интерпретация
Декларативная интерпретация: Результатом удаления любого элемента из пустого списка является пустой список. Если элемент, который мы удаляем, равен первому элементу списка (голове), то результирующий список равен результату удаления этого элемента из хвоста списка. Если удаляемый элемент не равен голове списка, то голова переходит в результирующий список, а остальная часть результата формируется из хвоста.

Процедурная интерпретация: Для удаления элемента X из списка необходимо проверить: если список пуст — вернуть пустой список. Иначе сравнить X с первым элементом. Если они равны, отбросить первый элемент и рекурсивно обработать хвост. Если не равны, сохранить первый элемент, рекурсивно обработать хвост и прикрепить сохраненный элемент к началу полученного результата.

## Задание 1. Удаление всех вхождений заданного элемента из списка
*(Вариант индивидуального задания №3)*

### Тестирование программы и результаты
Ниже приведены запросы, которые я выполняла для проверки корректности алгоритма:

**1. Обычный случай (элемент в разных частях списка):**
Запрос: `?- delete_all(a,[a, b, c, a, d, a], L)`
Результат: `L =[b, c, d]`
**Объяснение: Алгоритм успешно прошел по списку и отбросил все элементы, совпадающие с a.**

**2. Элемента нет в списке:**
Запрос: `?- delete_all(x, [a, b, c], L).`
Результат: `L = [a, b, c].`
**Объяснение: Ни одна голова списка не совпала с x, поэтому все элементы были перенесены в результирующий список без изменений.**

**3. Список состоит только из удаляемых элементов:**
Запрос: `?- delete_all(1, [1, 1, 1], L).`
Результат: `L =[].`
**Объяснение: Алгоритм отбросил каждую единицу, дошел до пустого списка и вернул его как итоговый результат.**

**4. Удаление из пустого списка (краевой случай):**
Запрос: `?- delete_all(a,[], L).`
Результат: `L =[].`
**Объяснение: Сработал базовый случай (первый факт программы).**

**5. Проверка истинности утверждения:**
Запрос: `?- delete_all(a, [a, b, a, c],[b, c]).`
Результат: `true.`
**Объяснение: Prolog подтвердил, что если из первого списка удалить a, то действительно получится список [b, c].**

## Задание 2. Подсчет количества элементов списка
*(Вариант индивидуального задания №9)*

##Интерпретация
Декларативная интерпретация: Длина пустого списка равна нулю. Длина списка, состоящего из головы и хвоста, равна N, где N — это длина хвоста плюс один.
Процедурная интерпретация: Чтобы найти длину списка, нужно проверить, пуст ли он. Если да, то результат равен 0. Иначе нужно отделить первый элемент от хвоста, найти длину хвоста (вызвав процедуру рекурсивно), прибавить к полученному значению 1 и выдать это как итоговый результат.

## Тестирование программы и результаты
**1. Обычный список:**
Запрос: `?- count_elements([a, b, c, d, e], N).`
Результат: `N = 5.`
**Объяснение: Рекурсия спустилась до пустого списка (0), а на обратном пути 5 раз прибавила по единице за каждый элемент.**

**2. Пустой список (краевой случай):**
Запрос: `?- count_elements([], N).`
Результат: `N = 0.`
**Объяснение: Сработал базовый случай напрямую.**

**3. Список из одного элемента:**
Запрос: `?- count_elements([x], N).`
Результат: `N = 1.`
**Объяснение: Голова отделилась, хвост оказался пустым (0). 0 + 1 = 1.**

**4. Список с вложенными списками:**
Запрос: `?- count_elements([1,[2, 3, 4], 5, [6, 7]], N).`
Результат: `N = 4.`
**Объяснение: Алгоритм корректно считает только элементы верхнего уровня. Вложенные списки [2, 3, 4] и [6, 7] воспринимаются как единичные элементы, так как мы не используем глубокую рекурсию.**

**5. Проверка ложного утверждения:**
Запрос: `?- count_elements([a, b, c], 5).`
Результат: `false.`
**Объяснение: Фактическое количество элементов списка равно 3, поэтому система отвергла утверждение о том, что их 5.**
